Prework Study Guide

HTML

  • The head element contains informationa about the webpage that is mostly not visible to the user.
  • The body element represents the visible content shown to the user.
  • Headings are elements used to separate content inside of the body to make it more accessible for the user.
  • Lists are made using the ul element with li elements being used for individual list items.
  • Semantic elements are elements that describe the element's purpose.
  • Non-Semantic elements are elements that are abreviated or do not describe the element's purpose.

CSS

  • You can style an element directly in HTML using inline CSS
  • You can emed an entire stylesheet directly into an HTML file using an internal style sheet that contains CSS rules
  • The most popular way to style a webpage is with an external CSS style sheet
  • A margin indicates how much space is around the outside of an element
  • A padding indicates how much space is around the content inside an element
  • A period designates a class selector, and applies CSS to any HTML element assigned to that class

Git

  • git status: checks the branch that you are currently on
  • git checkout -b branch-name: creates a new branch and switches to that branch
  • git add: adds files to staging
  • git commit: commit any files that have been added or changed, use -m to add a message to a commit
  • git push: uploads current branch from local repository to remote repository
  • git pull: updates current branch in local repository to match remote branch
